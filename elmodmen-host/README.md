# 🚀 𝑩𝑨𝑻𝑴𝑨𝑵 𝑫𝒆𝒗𓃠

بانل استضافة كامل (Hosting Control Panel) مبني بـ Flask — تسجيل دخول بنظام موافقة، إدارة سيرفرات (Python / Node.js)، مدير ملفات، كونسول، خطط، وبوت تيليجرام.

## ✨ المميزات
- 🔐 تسجيل / دخول مع **نظام موافقة الأدمن** (pending / approved / rejected)
- 🔑 كلمات المرور مشفّرة (werkzeug hashing)
- 🖥️ إدارة سيرفرات: تشغيل / إيقاف / إعادة / حذف + مراقبة تلقائية
- 📁 مدير ملفات كامل (رفع / تعديل / حذف / فك ضغط ZIP)
- 📊 خطط (Plans) + ترقية + API Keys
- 🤖 بوت تيليجرام للتحكم
- 👑 لوحة أدمن كاملة

## ⚙️ الإعداد (Environment Variables)
| المتغير | الوصف | افتراضي |
|---|---|---|
| `SECRET_KEY` | مفتاح تشفير الجلسات | (غيّره!) |
| `ADMIN_USER` | اسم الأدمن | `admin` |
| `ADMIN_PASS` | كلمة مرور الأدمن | `admin123` |
| `BOT_TOKEN` | توكن بوت تيليجرام (اختياري) | فارغ |
| `ADMIN_TELEGRAM_ID` | آي دي الأدمن للإشعارات | فارغ |
| `API_BASE_URL` | رابط البانل (للبوت) | `http://127.0.0.1:5000` |

## ▶️ التشغيل محليًا
```bash
pip install -r requirements.txt
python app.py
# افتح http://127.0.0.1:5000
# أدمن افتراضي: admin / admin123
```

## ☁️ الرفع على VPS (موصى به)
فكرة تشغيل سيرفرات المستخدمين تحتاج **VPS حقيقي** (Contabo / Hetzner / DigitalOcean) وليس استضافة مجانية مؤقتة، لأن:
- التخزين دائم (الحسابات والسيرفرات ما تتمسحش).
- كل البورتات متاحة (سيرفرات المستخدمين تبقى موصولة).
- تحكم كامل في الموارد.

```bash
# على VPS بـ Ubuntu
sudo apt update && sudo apt install -y python3-pip nodejs npm
pip install -r requirements.txt
gunicorn app:app --bind 0.0.0.0:5000 --workers 1 --threads 8 --timeout 120
```

> ⚠️ **ملحوظة:** `--workers 1` مهم جدًا لأن حالة السيرفرات محفوظة في الذاكرة. أكثر من worker يسبّب تضارب في تتبع العمليات.

## 🔒 حماية الكود قبل الرفع

> ⚠️ ملاحظة صريحة: مفيش "تشفير كامل" لكود شغّال — أي سيرفر/متصفح لازم يفك الكود عشان يشغّله. اللي بنعمله هنا **تعمية (Obfuscation) + حماية عملية** بتصعّب السرقة جدًا.

### الطريقة الأولى: سكربت جاهز (محلي)
```bash
bash scripts/build_protect.sh
```
هيعمل:
- **الباك إند** → يتحوّل لبايت كود محمي بـ PyArmor (مفيش `.py` مقروء).
- **الفرونت إند** → جافاسكربت مصغّر ومعمّى بـ terser.
- الناتج في مجلد `dist/` — ارفع محتوياته فقط.

### الطريقة الثانية: Docker (مناسب لـ Railway)
في Railway اختر **Deploy from Dockerfile** — الـ `Dockerfile` المرفق بيعمّي الكود تلقائيًا وقت البناء.

### نصائح حماية مهمة
1. خلّي مستودع الكود (GitHub repo) **Private** — ده أهم حاجة.
2. كل الأسرار في **Environment Variables** مش في الكود: `SECRET_KEY`, `ADMIN_USER`, `ADMIN_PASS`, `BOT_TOKEN`.
3. القوالب فيها **حماية خفيفة ضد النسخ** (قفل كليك يمين / F12 / عرض المصدر) كرادع.

### بخصوص المنصات
| المنصة | الباك إند (Flask) | ملاحظة |
|---|---|---|
| **Railway** | ✅ يشتغل | استخدم Dockerfile للحماية. التخزين مؤقت. |
| **Netlify** | ❌ | static/serverless فقط — مش هيشغّل Flask دائم. |
| **Vercel** | ❌ | static/serverless فقط — مش هيشغّل Flask دائم. |

> 💡 الواجهة (HTML/JS) دايمًا بتتحمّل في متصفح المستخدم، فمستحيل تتخفي 100% — التعمية بتخليها صعبة القراءة بس مش مخفية. الحماية الحقيقية مكانها الباك إند + الأسرار في environment.
